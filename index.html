How it works

Three layers.
Total visibility.

ai-obs is a pipeline from your AI code to your Grafana dashboard. Each layer is independently useful and stays completely out of your way.

Layer 01 — Your code
🎯
Instrument with one decorator

Add @observe(model=..., provider=...) to any function that calls an LLM. Zero changes to your existing logic. Also works as FastAPI middleware (instruments every route automatically) and a LangChain callback handler.

Layer 02 — The collector
📡
Events batch to a collector service

The SDK queues events in a background thread and flushes every 5 seconds — zero latency impact on your actual LLM calls. The FastAPI collector validates, persists to PostgreSQL, and updates Redis real-time counters in a single batch pipeline.

Layer 03 — Grafana
📊
Dashboards auto-populate on first boot

Three Grafana dashboards are provisioned automatically — no manual setup, no JSON import. Run docker compose up, open localhost:3000, and your latency, cost, accuracy, and error panels are already live with seed data.

Layer 04 — Plugins (optional)
🔌
Export to any system you already use

Register custom exporters to forward every event to Datadog, ClickHouse, S3, Slack, or any system in under 10 lines. Built-ins: WebhookExporter, S3Exporter, DatadogExporter, FileExporter. One failure never blocks others.

Providers

Every major LLM.
One SDK.

Token extraction and cost calculation built-in for all providers. Add a new one in 15 minutes with a single handler function and one call to register_provider().

from ai_obs import observe from openai import OpenAI client = OpenAI() @observe(model="gpt-4o", provider="openai", endpoint="chat") def ask(prompt: str) -> str: resp = client.chat.completions.create( model="gpt-4o", messages=[{"role": "user", "content": prompt}] ) return resp.choices[0].message.content # Captures: latency, prompt_tokens, completion_tokens, cost_usd, errors
from ai_obs import observe import anthropic client = anthropic.Anthropic() @observe(model="claude-sonnet-4-6", provider="anthropic") def ask_claude(prompt: str) -> str: msg = client.messages.create( model="claude-sonnet-4-6", max_tokens=1024, messages=[{"role": "user", "content": prompt}] ) return msg.content[0].text
from ai_obs import observe import boto3, json bedrock = boto3.client("bedrock-runtime", region_name="us-east-1") @observe(model="amazon.titan-text-express-v1", provider="bedrock") def ask_bedrock(prompt: str) -> str: resp = bedrock.invoke_model( modelId="amazon.titan-text-express-v1", body=json.dumps({"inputText": prompt}) ) return json.loads(resp["body"].read())["results"][0]["outputText"] # Also works: Claude-on-Bedrock, Llama-on-Bedrock, Mistral-on-Bedrock
from ai_obs import observe from huggingface_hub import InferenceClient client = InferenceClient() @observe(model="meta-llama/Llama-3-8B-Instruct", provider="huggingface") def ask_hf(prompt: str) -> str: return client.text_generation(prompt, max_new_tokens=512) # Works with Inference API, Inference Endpoints, and local pipelines
from ai_obs import observe import requests # provider="ollama" also works — it's an alias for generic @observe(model="llama3:8b", provider="ollama", endpoint="local") def ask_local(prompt: str) -> str: resp = requests.post( "http://localhost:11434/api/generate", json={"model": "llama3", "prompt": prompt, "stream": False} ) return resp.json()["response"] # Same pattern works for vLLM, LM Studio, any OpenAI-compatible server
from fastapi import FastAPI from ai_obs.middleware import AIObsMiddleware app = FastAPI() # Instruments EVERY route automatically — no per-route changes needed app.add_middleware( AIObsMiddleware, provider="openai", model="gpt-4o", skip_paths=["/health", "/docs", "/redoc"] ) # Every request gets an X-AI-Obs-Trace-Id header for manual scoring
from langchain_openai import ChatOpenAI from ai_obs.middleware import AIObsCallbackHandler # Works with any LangChain LLM, chain, or agent llm = ChatOpenAI( model="gpt-4o", callbacks=[AIObsCallbackHandler( provider="openai", tags={"framework": "langchain"} )] ) # All LangChain calls through this LLM are now tracked

Use cases

Who uses ai-obs
and why.

The same questions come up every time an AI system reaches production — regardless of team size or company stage.

Solo developers & indie hackers
Stop guessing how much your side project costs

You're shipping an LLM-powered app on a budget. You need to know exactly what each feature costs per call before you hit a surprise $500 bill at month end.

  • Real-time cost per model and endpoint
  • Which prompts are failing and why
  • Latency spikes that slow your app down
  • Usage growth as your user base scales
ML engineers & AI teams
Catch model quality degradation before users do

You've shipped a model to production. Now you need to know if accuracy is drifting, latency is creeping, or a new prompt version broke something quietly.

  • Accuracy drift detection with daily trends
  • LLM-as-judge auto-scoring at scale
  • p50/p95/p99 latency per model version
  • Error pattern analysis and rate tracking
Platform & DevOps teams
Treat AI APIs like any other production service

Your organisation runs multiple AI workloads. You need the same SLOs, alerting, and cost attribution you'd apply to any other microservice in your fleet.

  • SLO dashboards for latency and availability
  • Cost attribution per team or feature tag
  • Rate limit and quota hit tracking
  • Export to existing Datadog or PagerDuty
Built to be forked

Extend it.
Make it yours.

Every part of ai-obs was designed to be a starting point. Fork it, add your provider, add your exporter, ship it to your team in an afternoon.

~15 min
Add a new LLM provider

Write one handler function in sdk/ai_obs/providers/ that extracts token counts from the response object. Add cost table entries. Register it. Done.

register_provider("myprovider", handler)
~10 min
Write a custom exporter

Subclass BaseExporter in sdk/ai_obs/exporters/. Implement one method: export(self, events). Register it. Every batch flows to your system.

registry.register(MyExporter())
~20 min
Add a Grafana panel

Drop a new panel into any JSON file in dashboard/provisioning/dashboards/. Grafana hot-reloads provisioned dashboards every 30 seconds — no restart needed.

dashboard/provisioning/dashboards/
~30 min
New framework integration

Add to sdk/ai_obs/middleware/ following the FastAPI middleware or LangChain callback handler patterns already there. Both are under 120 lines.

sdk/ai_obs/middleware/__init__.py
~15 min
New collector endpoint

Add a router file in collector/app/routers/ and include it in main.py. PostgreSQL schema changes go in storage/migrations/ as plain SQL.

collector/app/routers/myfeature.py
~5 min
Publish your Docker image

The CI pipeline auto-publishes to Docker Hub on every GitHub Release. Set DOCKERHUB_USERNAME and DOCKERHUB_TOKEN as repo secrets and push a tag.

.github/workflows/ci.yml
Quick start

Up and running
in 60 seconds.

No account. No credit card. No YAML configuration files. Just Docker and Python.

Step 1 — Stack
Clone & start
git clone github.com/your-org/ai-obs cd ai-obs cp .env.example .env docker compose up -d

PostgreSQL, Redis, MinIO, Collector and Grafana all start together. Seed data loads automatically so dashboards look populated immediately.

Step 2 — SDK
Install
pip install ai-obs-sdk # With provider extras: pip install ai-obs-sdk[openai] pip install ai-obs-sdk[anthropic] pip install ai-obs-sdk[bedrock]

Zero hard dependencies in the core SDK. Provider SDKs are optional extras.

Step 3 — Code
Add one decorator
from ai_obs import observe @observe( model="gpt-4o", provider="openai" ) def ask(prompt): ... # existing code unchanged

Wrap any function. Your existing logic does not change at all.

Step 4 — View
Open Grafana
open http://localhost:3000 # Login: admin / admin # API docs at: # http://localhost:8080/docs

Metrics appear within seconds of your first call. Three dashboards ready with no configuration needed.

All SDK settings via environment variables — sensible defaults, nothing required
AI_OBS_ENDPOINT          http://localhost:8080   URL of the collector
AI_OBS_API_KEY           (none)                 Set in production for auth
AI_OBS_ENV               production             Tag: production / staging / dev
AI_OBS_SAMPLE_RATE       1.0                    0.1 = capture 10% at high volume
AI_OBS_BATCH_SIZE        50                     Events per HTTP flush
AI_OBS_FLUSH_INTERVAL    5                      Seconds between background flushes
AI_OBS_DISABLE           false                  true = silent in unit tests
AI_OBS_CAPTURE_PROMPTS   false                  Opt-in: store prompt text in DB
AI_OBS_CAPTURE_COMPLETIONS false               Opt-in: store completion text in DB
